## 📋 Описание

Данный репозиторий содержит полный комплект **Фонда оценочных средств (ФОС)** для оценки проектной работы по дисциплине *«Методы оптимизации в системах искусственного интеллекта»*. 

Репозиторий разработан в соответствии с **Компетентностно-ролевой моделью (КРМ) ИТМО** и реализует принцип **«зеркальной» структуры**: он оформлен как программный продукт, повторяющий архитектуру интеллектуальной информационной системы (ИИС), которую создают студенты. Это позволяет использовать данный репозиторий не только как методический инструмент, но и как **эталонный референс** для студентов по инженерной документации, структуре проекта и качеству кода.

## 🎯 Целевая аудитория

- **Преподаватели и ассистенты:** для применения рубрик, шаблонов и проведения оценки (peer-review, защита).
- **Методическая комиссия ОП:** для аудита, приемки и верификации соответствия ФОС стандартам университета.
- **Студенты:** как образец (референс) ожидаемой структуры репозитория, качества документации и критериев оценки их собственных проектов.

## 🚀 Быстрый старт (для преподавателя)

Чтобы начать использовать этот ФОС в учебном процессе за 30 минут:

1. **Ознакомьтесь с паспортом:** Изучите [`docs/passport.md`](docs/passport.md) для понимания весов, шкал и порогов.
2. **Изучите критерии:** Откройте [`docs/project_criteria.md`](docs/project_criteria.md), чтобы понять требования к осям «MVP» и «Документация».
3. **Скачайте печатные формы:** Возьмите готовые PDF-шаблоны из папки [`release/printable/`](release/printable/) для использования на защитах.
4. **Прочтите инструкцию:** Следуйте пошаговому гайду [`release/how_to_apply.md`](release/how_to_apply.md) для интеграции ФОС в ваш учебный поток.

## ⚖️ Ключевые параметры оценивания

| Параметр | Значение |
| :--- | :--- |
| **Профильные роли КРМ** | `ML Researcher`, `AI Security Engineer` |
| **Целевые компетенции** | **MF-3** (Методы оптимизации), **ML-3** (Классическое МО с учителем), **ML-5** (Устойчивость и безопасность МО) |
| **Оси оценивания КП** | 1. Программное решение (MVP)<br>2. Документация<br>3. Соответствие индикаторам роли (сквозной) |
| **Шкала** | 10-балльная (с маппингом на 5-балльную: 8–10 «Отлично», 6–7 «Хорошо», 4–5 «Удовл.», <4 «Неуд.») |
| **Порог освоения** | **≥ 4.0** баллов в сумме И **≥ 4.0** баллов по каждой из осей оценивания |

## 🪞 Принцип «зеркальной» структуры

Структура этого репозитория напрямую отражает структуру репозитория ИИС, который должен сдать студент. Это обеспечивает содержательное соответствие и снижает когнитивную нагрузку при объяснении требований.

| Артефакт студенческой ИИС | Аналог в репозитории ФОС | Смысл зеркальности |
| :--- | :--- | :--- |
| `README.md` | [`README.md`](README.md) | Корневое описание, быстрый старт, целевая аудитория. |
| `PROJECT_BRIEF.md` | [`FOS_BRIEF.md`](FOS_BRIEF.md) | Постановка задачи: какая дисциплина, какие роли КРМ закрываются. |
| `docs/architecture.md` | [`docs/fos_architecture.md`](docs/fos_architecture.md) | Архитектура системы ↔ Архитектура оценивания и потоки данных КРМ. |
| `data/` | [`data/competency_map/`](data/competency_map/) | Данные системы ↔ Машиночитаемая выгрузка компетенций и индикаторов. |
| `src/models/` | [`src/rubrics/`](src/rubrics/) | Модели ИИ ↔ Математические/логические модели оценивания (рубрики). |
| `src/api/` | [`src/templates/`](src/templates/) | Интерфейсы системы ↔ Шаблоны оценочных листов (API для взаимодействия людей). |
| `tests/` | [`tests/`](tests/) | Unit/Integration тесты ↔ Скрипты и чек-листы валидности самого ФОС. |
| `experiments/` | [`validation/`](validation/) | Эксперименты с данными ↔ Пилотный прогон рубрик на реальных работах. |
| `deployment/` | [`release/`](release/) | Развертывание ↔ Инструкция по внедрению ФОС в учебный процесс. |

## ✅ Критерии приёмки (Статус: ВЫПОЛНЕНО)

Данный ФОС прошел внутреннюю проверку и соответствует всем критериям ТЗ:

- [x] **ПР-1. Полнота артефактов:** Все файлы обязательного минимума присутствуют.
- [x] **ПР-2. Соответствие КРМ:** Каждая заявленная компетенция имеет индикаторы и оценивается $\ge$ 2 средствами (триангуляция).
- [x] **ПР-3. Полнота 10-балльной шкалы:** Описаны уровни 10 / 8 / 6 / 4 / 0–3 для обеих осей.
- [x] **ПР-4. Ролевая проработка:** Для ролей *ML Researcher* и *AI Security Engineer* созданы портреты уровней и ролевые блоки в шаблонах.
- [x] **ПР-5. Согласованность весов:** Сумма весов ОС и внутри рубрик строго равна 100% (подтверждено в `tests/weights_check.md`).
- [x] **ПР-6. Валидация:** Проведен пилотный прогон на 3-х работах, пороги откалиброваны (`validation/report.md`).
- [x] **ПР-7. Воспроизводимость:** Сторонний преподаватель может применить ФОС, следуя `release/how_to_apply.md`, без консультаций с автором.
- [x] **ПР-8. Инженерная культура:** Осмысленная история коммитов, ADR, теги версий, `CHANGELOG.md`, чистый `README.md`.

## 🔄 Сопровождение и развитие

- **Минорные обновления** (v1.1, v1.2): Вносятся после каждого семестра на основе обратной связи (через Issues) и фиксируются в `CHANGELOG.md`.
- **Мажорные обновления** (v2.0): Требуются при смене профильных ролей КРМ, изменении весов оценочных средств или существенной переработке рубрик. Оформляются через новый ADR в `meta/decisions/`.
- **Обратная связь:** Замечания от преподавателей и студентов принимаются через создание Issue с шаблоном `methodological_issue.md`.
