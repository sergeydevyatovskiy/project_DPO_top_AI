# Ось «Документация» — 10-балльная рубрика

**Дисциплина:** Методы оптимизации в системах искусственного искусственного интеллекта  
**Вес оси:** 35% итоговой оценки курсового проекта

## Критерии оси

| № | Критерий | Вес внутри оси |
|---|----------|----------------|
| 1 | Репозиторная документация | 29% |
| 2 | Методология эксперимента | 29% |
| 3 | Анализ результатов и интерпретация | 29% |
| 4 | Анализ ограничений, рисков и практической ценности | 13% |

## Уровни оценивания

### 10 баллов — «Архитектор-визионер»

- Документация уровня публикации: полный набор артефактов (README, PROJECT_BRIEF, methodology.md, robustness_analysis.md, инструкции, демо-сценарии).
- Проект **полностью воспроизводим** внешним экспертом на чистом окружении (Docker/Conda).
- Методология эксперимента описана строго: гиперпараметры, условия сходимости, критерии остановки, версионирование данных.
- Анализ результатов содержит **связь теории оптимизации с практикой** (например, объяснение сходимости Adam через теорию выпуклости).
- Описаны границы применимости, вычислительные затраты, рекомендации по применению в индустрии.

### 8 баллов — «Проектировщик»

- Полный комплект документации; репозиторий структурирован; README информативен.
- Проект воспроизводим при следовании инструкции.
- Методология эксперимента описана: гиперпараметры, метрики, условия.
- Анализ результатов обоснован; выводы аргументированы.
- Описаны ограничения и направления развития.

### 6 баллов — «Самостоятельный»

- Документация присутствует в достаточном объёме: README, краткое описание методологии, инструкции запуска.
- Проект воспроизводим с небольшими уточнениями.
- Методология описана частично; гиперпараметры указаны.
- Анализ результатов присутствует, но частично поверхностный.
- Ограничения упомянуты.

### 4 балла — «Исполнитель»

- Минимальная документация: есть README и краткое описание, но воспроизведение возможно только с помощью автора.
- Методология не описана; гиперпараметры не зафиксированы.
- Анализ результатов формальный.
- Ограничения не описаны.

### 0–3 балла — «Не сформирован»

- Документация отсутствует или существенно неполна; проект невоспроизводим.
