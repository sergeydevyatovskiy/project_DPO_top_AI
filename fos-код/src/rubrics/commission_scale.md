# Уровни защиты 0–10 перед комиссией

**Дисциплина:** Методы оптимизации в системах искусственного интеллекта

## Расшифровка уровней шкалы для защиты

| Балл | Характеристика уровня защиты |
|------|------------------------------|
| **10** | Выдающаяся защита: чёткая логика, глубокое владение теорией оптимизации, живая демонстрация сходимости методов без замечаний, ответы уровня специалиста в отрасли, признаки научной новизны или высокой практической ценности. |
| **9** | Отличная защита: развёрнутые аргументированные ответы, качественная демонстрация, комиссия имеет только уточняющие вопросы. |
| **8** | Уверенная защита: полные и корректные ответы на вопросы о методах оптимизации, гиперпараметрах, устойчивости; качественные слайды; демонстрация проходит с минимальными замечаниями. |
| **7** | Хорошая защита: аргументированные ответы на большинство вопросов, отдельные пробелы в деталях реализации или теории сходимости. |
| **6** | Достаточная защита: основные ответы даны, ответы на глубокие вопросы о робастности и adversarial-атаках неполные, демонстрация проходит по подготовленному сценарию. |
| **5** | Слабая защита: студент справляется только с ответами по подготовленному тексту, демонстрация ограничена. |
| **4** | Минимально приемлемая защита: демонстрирует базовое понимание методов оптимизации, испытывает затруднения с вопросами о сходимости, гиперпараметрах, устойчивости. |
| **3** | Ниже порога: серьёзные пробелы в понимании теории оптимизации и практики обучения моделей. |
| **2** | Фрагментарно: демонстрирует лишь отдельные элементы работы (например, только запуск кода без понимания). |
| **0–1** | Не защищена: не может воспроизвести собственный проект, не отвечает на существенные вопросы комиссии, демонстрация отсутствует. |
